Give these samples a try to see various ways of using the SeekU runtime.

Samples include:

+ In memory sample - simplest possible demonstration with all execution and persistance in memory.
+ Text file sample - all events and snapshots are stored as json inside text file with a .json extension.
+ Sql sample - all events and snapshots are stored in SQL tables as json blobs.
+ NServiceBus sample - shows how commands can be sent through NServiceBus from client to server (requires 2 startup projects).
+ Azure sample - shows how commands can be sent through Azure queues
